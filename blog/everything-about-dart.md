# DART의 모든 것

DART를 처음 접하는 사람에게도, 이미 쓰고 있지만 전체 그림을 한 번에 보고 싶은 사람에게도 이 글이 도움이 되길 바란다.

## DART란?

DART(**D**ata **A**nalysis, **R**etrieval and **T**ransfer System)는 **금융감독원**이 운영하는 전자공시시스템이다. 한국 자본시장에서 공개적으로 거래되는 기업이 법적으로 제출해야 하는 모든 공시 서류를 전자적으로 제출하고, 누구나 조회할 수 있게 공개하는 플랫폼이다.

1999년 처음 도입된 이후 한국 자본시장 투명성의 핵심 인프라로 자리잡았다. 공식 주소는 [dart.fss.or.kr](https://dart.fss.or.kr)이다.

DART가 존재하기 전에는 공시 서류를 직접 금융감독원이나 거래소에 방문하거나, 종이 서류를 우편으로 받아야 했다. DART 덕분에 누구든 인터넷만 있으면 상장기업의 모든 공시를 즉시 열람할 수 있게 되었다.

## 누가 운영하는가

**금융감독원**(FSS, Financial Supervisory Service)이 운영한다. 금융감독원은 은행, 증권, 보험, 자본시장 등 금융 전반을 감독하는 기관이다. DART는 그 감독 기능의 일부로, 상장기업의 정보 공개 의무를 관리하는 창구다.

상장기업은 사업보고서, 반기보고서, 분기보고서 등을 법정 기한 내에 DART를 통해 제출할 의무가 있다. 제출하지 않으면 행정 제재를 받는다. 이 의무는 자본시장과 금융투자업에 관한 법률(자본시장법)에 근거한다.

금융감독원은 제출된 공시를 검토하고, 부실 기재가 발견되면 정정을 요구하거나 제재를 부과한다.

## 무엇이 공시되는가

공시 유형은 크게 세 가지로 나뉜다.

### 정기공시

가장 중요하고 분량이 많다. 기업의 재무 상태, 경영 현황, 리스크를 종합적으로 파악할 수 있는 기본 자료다.

| 보고서 | 제출 기한 | 주요 내용 |
|--------|-----------|-----------|
| **사업보고서** | 사업연도 종료 후 90일 | 재무제표 전체, 사업 내용, 임원/주주/자회사 현황, 감사의견 |
| **반기보고서** | 반기 종료 후 45일 | 사업보고서의 축약 버전, 반기 재무제표 |
| **분기보고서** | 분기 종료 후 45일 | 1분기, 3분기 재무제표 위주 |

사업보고서가 가장 상세하다. 200~300페이지에 달하는 경우도 흔하다. 기업 분석의 출발점이 되는 문서다.

### 주요사항보고서

중요 경영 이벤트 발생 시 즉시 제출하는 수시 공시다.

- **자금 조달**: 유상증자, 무상증자, 전환사채(CB) 발행, 신주인수권부사채(BW) 발행
- **지배구조 변경**: 최대주주 변경, 임원 선임/해임
- **소송 및 분쟁**: 대규모 소송 제기/판결
- **구조 변경**: 합병, 분할, 영업양수도, 해산

시장에 즉각적인 영향을 줄 수 있는 사건들이므로, 공시 지연 자체가 제재 사유가 된다.

### 자율공시

법적 의무는 없지만 기업이 자발적으로 제출하는 공시다. 실적 전망, 신제품 출시, MOU 체결, 공장 신설 등이 해당한다. 기업 IR(Investor Relations) 활동의 일환으로 활용된다.

## 공시 문서의 구조

사업보고서를 예로 들면, 한 문서가 수십 개의 섹션으로 구성된다. 각 섹션은 서로 다른 종류의 정보를 담고 있다.

| 섹션 | 주요 내용 | 데이터 유형 |
|------|-----------|-------------|
| **회사의 개요** | 상호, 설립일, 주요 사업, 임직원 수 | 정량 + 텍스트 |
| **사업의 내용** | 제품/서비스 설명, 시장 환경, 경쟁 현황 | 텍스트 |
| **재무에 관한 사항** | 요약재무정보, 재무제표 본문, 주석 | 정량 |
| **감사인의 감사의견** | 적정/한정/부적정/의견거절, 핵심감사사항 | 텍스트 |
| **이사의 경영진단** | MD&A — 경영진이 직접 서술하는 실적 설명 | 텍스트 |
| **임원 및 직원 현황** | 임원 보수, 직원 수, 평균 급여, 근속 연수 | 정량 |
| **최대주주 현황** | 최대주주 및 특수관계인 지분율 변동 | 정량 |
| **주주에 관한 사항** | 배당 내역, 자기주식 현황, 소액주주 | 정량 |
| **이사회 등** | 이사회 구성, 사외이사 비율, 출석률 | 정량 + 텍스트 |
| **계열회사 등** | 자회사/관계회사 현황, 내부거래 | 정량 |

이 중 "재무에 관한 사항"이 숫자 분석의 핵심이고, "사업의 내용"과 "감사의견"이 텍스트 분석의 핵심이다.

### 요약재무정보

사업보고서 첫 부분에 나오는 최근 3~5개년 주요 재무지표 요약이다. 매출액, 영업이익, 당기순이익, 자산총계, 부채총계, 자본총계 등이 한 표에 정리되어 있다.

이 요약표가 재무 시계열 분석의 출발점이다. 다만, 매년 계정명이 미세하게 바뀌는 문제가 있다. 이것은 뒤에서 자세히 다룬다.

### 재무제표 본문

요약이 아닌 전체 재무제표다. 재무상태표, 포괄손익계산서, 현금흐름표, 자본변동표로 구성된다. 주석(Notes)이 포함되어 계정별 상세 내역을 확인할 수 있다.

### 감사보고서

외부 감사인(회계법인)이 재무제표를 검토한 결과를 담은 보고서다. 감사의견은 4가지로 분류된다.

| 의견 | 의미 |
|------|------|
| **적정** | 재무제표가 회계기준에 따라 적정하게 작성됨 |
| **한정** | 일부 항목을 제외하면 적정 |
| **부적정** | 중요한 왜곡표시가 있음 |
| **의견거절** | 감사 범위 제한으로 의견 표명 불가 |

대부분의 상장기업은 적정 의견을 받는다. 적정이 아닌 의견은 그 자체로 강력한 리스크 시그널이다.

## K-IFRS와 재무제표

한국 상장기업은 2011년부터 **K-IFRS**(한국채택국제회계기준)를 의무 적용한다. K-IFRS는 IFRS(국제회계기준)를 한국에 맞게 채택한 것으로, 글로벌 투자자가 한국 기업의 재무제표를 이해할 수 있게 하는 기반이다.

### 연결 vs 별도

재무제표는 **연결재무제표**와 **별도재무제표** 두 가지로 제출한다.

- **연결(Consolidated)**: 지배회사 + 종속회사를 하나의 경제적 실체로 보고 합산한다. 투자자 관점에서 더 중요하다. 그룹 전체의 실적과 재무 상태를 보여준다.
- **별도(Separate)**: 지배회사 단독 재무제표다. 종속회사 투자를 원가법이나 지분법으로 처리한다.

분석 시에는 보통 연결재무제표를 기준으로 한다.

### 재무제표의 4가지 구성요소

| 명칭 | 영문 약자 | 핵심 질문 |
|------|-----------|-----------|
| **재무상태표** | BS (Balance Sheet) | 기업이 무엇을 가지고 있고, 얼마나 빌렸는가? |
| **포괄손익계산서** | IS (Income Statement) | 얼마나 벌고, 얼마나 썼는가? |
| **현금흐름표** | CF (Cash Flow) | 실제 현금은 어디서 왔고, 어디로 갔는가? |
| **주석** | Notes | 각 계정의 세부 내역은 무엇인가? |

#### 재무상태표 (BS)

특정 시점의 재무 상태를 보여준다. 핵심 등식은 **자산 = 부채 + 자본**이다.

- **자산**: 현금, 매출채권, 재고자산, 유형자산, 무형자산 등
- **부채**: 매입채무, 차입금, 사채, 충당부채 등
- **자본**: 자본금, 이익잉여금, 기타포괄손익누계액 등

#### 포괄손익계산서 (IS)

일정 기간의 경영 성과를 보여준다.

```
매출액 (Revenue)
 - 매출원가 (COGS)
= 매출총이익 (Gross Profit)
 - 판매비와관리비 (SG&A)
= 영업이익 (Operating Income)
 ± 영업외수익/비용
= 법인세비용차감전순이익
 - 법인세비용
= 당기순이익 (Net Income)
```

#### 현금흐름표 (CF)

실제 현금의 흐름을 세 가지로 분류한다.

- **영업활동 현금흐름**: 본업에서 발생한 현금. 양수여야 건전하다.
- **투자활동 현금흐름**: 설비투자, 인수합병 등. 보통 음수다.
- **재무활동 현금흐름**: 차입, 상환, 배당, 증자 등.

#### 주석 (Notes)

재무제표 각 계정의 상세 내역이다. 재고자산의 구성, 차입금의 만기 구조, 유형자산의 감가상각 내역, 관계기업 투자 현황 등을 확인할 수 있다.

K-IFRS는 주석 공시 항목을 상세하게 규정하고 있어, 주석만으로도 기업의 재무 구조를 깊이 파악할 수 있다.

## DART Open API

DART는 [Open API](https://opendart.fss.or.kr)도 제공한다. API 인증키를 발급받으면 프로그래밍으로 공시 데이터에 접근할 수 있다.

### 주요 API

| API | 내용 |
|-----|------|
| 공시검색 | 기간별, 기업별 공시 목록 조회 |
| 사업보고서 | 재무제표, 배당, 임원, 주주 등 정형 데이터 |
| 재무정보 | 단일회사 또는 다중회사 재무제표 |
| 지분공시 | 임원·주요주주 소유 보고 |
| 공시서류 원문 | XBRL, HTML 원문 다운로드 |

### API의 한계

DART Open API는 유용하지만, 몇 가지 한계가 있다.

- **정형 데이터만 제공**: 사업의 내용, 감사의견 등 텍스트 섹션은 API로 구조화되어 제공되지 않는다.
- **항목 한정**: 주석 데이터는 일부만 API로 제공되고, 나머지는 원문을 직접 파싱해야 한다.
- **계정명 비정규화**: 같은 계정이라도 기업마다, 연도마다 다른 이름으로 표기된다.
- **호출 제한**: 일일 호출 횟수 제한이 있다.

## 왜 자동화가 필요한가

DART는 방대한 데이터를 제공하지만, 그대로 분석에 쓰기 어렵다. 이유는 크게 세 가지다.

### 1. 계정명이 매년 바뀐다

K-IFRS 개정이나 기업 내부 정책 변경으로 동일한 계정의 이름이 바뀐다.

```
2022년: 매출액
2023년: 매출액
2024년: 수익(매출액)
```

"매출액"과 "수익(매출액)"은 같은 계정이지만, 단순 문자열 비교로는 다른 항목으로 인식된다. 시계열을 만들려면 이 변동을 추적해야 한다.

이런 변경은 한 기업 안에서도, 기업마다도 다르게 일어난다. 수작업으로 매핑하면 수백 개 기업의 수십 년 치 데이터를 일일이 확인해야 한다.

### 2. 분기 누적값 문제

분기보고서와 반기보고서의 손익계산서는 **연초부터의 누적값**이다.

```
                       1분기      2분기      3분기      4분기
                      ┌──────┐
 1분기 보고서          │  Q1  │
                      └──────┘
                      ┌──────────────┐
 반기 보고서           │   Q1 + Q2    │
                      └──────────────┘
                      ┌─────────────────────┐
 3분기 보고서          │    Q1 + Q2 + Q3     │
                      └─────────────────────┘
                      ┌──────────────────────────────┐
 사업 보고서           │       Q1 + Q2 + Q3 + Q4      │
                      └──────────────────────────────┘
```

개별 분기 값(예: 3분기만의 매출)을 얻으려면 역산이 필요하다. 3분기 보고서의 Q1+Q2+Q3에서 반기 보고서의 Q1+Q2를 빼면 Q3만 남는다.

이 과정에서 연결/별도 구분, 비교 기간 처리, 계정명 변동까지 고려해야 해서 수작업으로는 매우 번거롭다.

### 3. 텍스트 파싱의 어려움

감사의견, 핵심감사사항, 사업의 내용, 위험 요인 등은 사람이 읽도록 작성된 **자유 서식 텍스트**다. 기업마다 서술 방식이 다르고, 같은 기업도 연도별로 구성이 바뀐다.

예를 들어 감사의견에서 "적정 / 한정 / 부적정 / 의견거절"을 자동 분류하려면, 다양한 표현 패턴을 인식하는 규칙 기반 파싱이 필요하다.

사업의 내용에서 특정 섹션(예: "위험 요인")만 추출하려면 문서 구조를 분석해야 하는데, 이 구조가 기업마다 다르다.

## DartLab이 해결하는 방식

DartLab은 위의 세 가지 문제를 각각 해결하는 36개 분석 모듈의 집합이다.

### Bridge Matching — 계정명 추적

인접 연도의 재무제표에서 **금액과 명칭 유사도**를 조합해 동일 계정을 자동으로 연결한다.

```
             2022년            2023년            2024년
             ──────            ──────            ──────
 매출액 ────────────── 매출액 ────────────── 수익(매출액)
 영업이익 ──────────── 영업이익 ──────────── 영업이익
 당기순이익 ────────── 당기순이익 ────────── 당기순이익(손실)
```

4단계 매칭으로 구성된다.

1. **정확 매칭**: 금액이 완전히 동일한 계정 연결
2. **재작성 매칭**: 소수점 오차(0.5 이내) 허용
3. **명칭 변경 매칭**: 금액 오차 5% 이내 + 명칭 유사도 60% 이상
4. **특수 항목 매칭**: 주당이익(EPS) 등 소수점 단위 항목

매칭률이 85% 이하로 떨어지면 변경점(breakpoint)으로 판정하고 구간을 분리한다.

### property 기반 API

모든 데이터를 Python property로 바로 접근할 수 있다.

```python
from dartlab import Company

c = Company("005930")    # 삼성전자

# 재무제표
c.BS    # 재무상태표 DataFrame
c.IS    # 손익계산서 DataFrame
c.CF    # 현금흐름표 DataFrame

# 정기보고서
c.dividend        # 배당 시계열
c.majorHolder     # 최대주주 지분율
c.employee        # 직원 현황
c.audit           # 감사의견

# K-IFRS 주석
c.notes.inventory          # 재고자산
c.notes.tangibleAsset      # 유형자산
c.notes.borrowings         # 차입금
c.notes["매출채권"]         # 한글 키도 가능

# 공시 텍스트
c.business        # 사업의 내용
c.mdna            # 경영진단의견
c.overview        # 회사의 개요

# 전체 일괄 추출
d = c.all()       # 40개 모듈 전체를 dict로
```

### 공시 수평 정렬

분기/반기/사업보고서를 하나의 시간축 위에 정렬하고, 누적값에서 개별 분기를 역산한다.

### K-IFRS 주석 통합

12개 주요 주석 항목을 `c.notes`로 통합 접근하고, 추가 23개 키워드는 `c.get("notesDetail", keyword="법인세")` 패턴으로 조회한다.
